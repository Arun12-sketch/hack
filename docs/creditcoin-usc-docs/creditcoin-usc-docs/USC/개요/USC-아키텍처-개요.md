# USC 아키텍처 개요

## 개요

*탈중앙화 오라클*은 블록체인 외부의 정보를 제공하는 시스템으로, 보안을 위해 중앙화된 신뢰 기관에 의존하지 않습니다. 대신 오라클의 데이터 제공 과정의 각 단계는 독립적인 행위자 그룹에 의해 실행되며, 이들 중 어느 누구도 데이터 제공 결과에 일방적으로 간섭할 수 없습니다. Creditcoin에 탈중앙화 데이터 제공 기능을 추가함으로써, *모든* 블록체인의 상태에 접근할 수 있는 스마트 컨트랙트를 신뢰 없이(trustlessly) 지원할 수 있게 됩니다. **유니버설 스마트 컨트랙트(Universal Smart Contracts, USCs)**로 알려진 이 스마트 컨트랙트들은 다른 체인에서 검증된 이벤트에 반응할 수 있는 새로운 크로스체인 애플리케이션을 가능하게 합니다.

## 주요 용어

- **소스 체인(Source chain):** Creditcoin이 데이터 제공을 지원하는 클라이언트 체인입니다. 먼저 `Ethereum`과 같은 `EVM` 체인을 지원하는 것으로 시작하며, 궁극적으로는 모든 체인에서 데이터 제공을 가능하게 하는 것이 목표입니다.

- **실행 체인(Execution chain):** 오라클 인프라와 유니버설 스마트 컨트랙트를 지원하는 Creditcoin 블록체인입니다.

- **증명(Attestation):** 합의를 통해 검증된 소스 체인 블록의 데이터에 대한 암호학적 커밋먼트입니다.

- **쿼리(Query):** 트랜잭션을 검증하기 위한 요청입니다. 각 쿼리는 검증하고자 하는 *소스 체인*, *블록 번호*, *트랜잭션*을 지정합니다. 그런 다음 USC는 검증된 트랜잭션 바이트에서 필요한 데이터를 추출합니다.

- **증거(Proof):** 특정 트랜잭션의 일부가 *소스 체인*에서 발생했음을 인증하는 암호학적 증거입니다.

> 증거는 머클 증거(트랜잭션 포함 여부)와 연속성 증거(블록체인 무결성)로 구성됩니다. 이들은 Creditcoin에서 네이티브 속도로 검증됩니다. 시스템은 특수한 증명 시스템의 필요 없이 검증된 트랜잭션 바이트에서 직접 트랜잭션 데이터를 추출합니다.

- **네이티브 증거 검증기 프리컴파일(Native Proof Verifier Precompile):** 쿼리를 네이티브 속도로 검증하는 Creditcoin의 네이티브 프리컴파일(주소 `0x0FD2`)입니다. 머클 증거와 연속성 체인을 사용하여 트랜잭션 포함 및 블록 포함을 검증합니다.

> [!danger]
> 네이티브 검증기 프리컴파일은 트랜잭션이 성공했는지 여부를 검증하지 **않습니다**. 트랜잭션이 블록에 포함되었는지와 해당 블록이 실제로 확인된 소스 체인의 일부인지만 검증합니다. 따라서 트랜잭션의 성공 여부는 DApp 개발자가 유니버설 스마트 컨트랙트에서 검증해야 합니다.

## 아키텍처

Creditcoin 탈중앙화 오라클은 주로 다음 행위자들에 의존합니다:

### 증명자(Attestors)

증명자들은 Ethereum과 같은 소스 체인의 최신 상태에 대한 자신들의 관점을 주장합니다. Creditcoin은 소스 체인 상태 변경에 대한 단일 증명자의 보고를 신뢰하지 않습니다. 대신, 탈중앙화된 증명자 네트워크가 어떤 상태 변경이 발생했는지(있다면)에 대해 합의에 도달해야 합니다. 이 합의는 Creditcoin 검증자가 검증할 수 있는 개별 증명자 투표의 집계 서명으로 제공됩니다.

> 증명자에 대한 자세한 정보는 문서의 [[USC/Creditcoin-Oracle-서브시스템/증명|증명(attestation)]] 섹션을 확인하세요.

### 검증자(Validators)

검증자들은 Creditcoin 블록체인의 권한 집합을 구성합니다. 검증자는 증명 트랜잭션을 수신하고, 기본적인 구조 검사를 수행하며, 합의를 통해 블록에 포함시킵니다. 런타임(검증자에 의해 실행됨)은 증명자 BLS 서명을 검증하고 증명을 온체인 저장소에 커밋하기 전에 충분한 정족수에 도달했는지 확인합니다.

### 네이티브 쿼리 검증기 프리컴파일

네이티브 쿼리 검증기 프리컴파일(*블록 증명자 프리컴파일*이라고도 함)은 Creditcoin 트랜잭션 내에서 크로스체인 데이터를 검증하는 주소 `0x0FD2`의 런타임 컴포넌트입니다. 이는 두 가지 증거를 검증합니다: 블록 내 트랜잭션 포함에 대한 머클 증거와, 블록 다이제스트 체인을 통해 해당 블록을 온체인 증명 또는 체크포인트에 연결하는 연속성 증거입니다.

프리컴파일은 컴파일된 Rust 코드로 실행되어 EVM 해석 오버헤드를 방지합니다. 검증은 동기적입니다: 트랜잭션 데이터, 머클 증거, 연속성 증거가 주어지면, 머클 루트가 연속성 체인의 블록과 일치하는지, 체인이 유효한 증명/체크포인트에서 끝나는지, 블록 다이제스트가 암호학적 해싱을 통해 올바르게 연결되어 있는지 확인합니다.

두 가지 함수를 사용할 수 있습니다: `verify()`(조회 전용, 이벤트 없음)와 `verifyAndEmit()`(상태 변경, `TransactionVerified` 이벤트 발생). USC 컨트랙트는 이를 사용하여 단일 트랜잭션에서 크로스체인 이벤트와 트랜잭션을 검증하며, 외부 증명 시스템과 오프체인 서비스를 대체합니다.

## 상호운용성

이 설계의 순 효과는 서드파티 개발자가 다른 체인의 검증된 데이터에 안전하고 신뢰 없이 접근할 수 있는 컨트랙트를 Creditcoin에서 생성할 수 있다는 것입니다. USC 컨트랙트는 외부 블록체인(예: Ethereum)에서 특정 트랜잭션이 발생했음을 검증한 다음, 검증된 이벤트에 반응하여 Creditcoin에서 비즈니스 로직을 실행할 수 있습니다.

예를 들어, 브릿지 컨트랙트는 다음을 수행할 수 있습니다:

1. 사용자가 Ethereum에서 ETH를 소각했는지 검증 (프리컴파일을 사용하여 소각 트랜잭션 검증)
2. 검증된 증거를 기반으로 Creditcoin에서 동등한 토큰 발행

개발자는 이러한 속성을 활용하여 자체 맞춤형 크로스체인 DApp 비즈니스 로직을 지원하는 유니버설 스마트 컨트랙트를 생성할 수 있으며, 중앙화된 오라클이나 중개자에 의존하지 않고 신뢰 없는 크로스체인 애플리케이션을 구현할 수 있습니다.

> Creditcoin `USC`를 활용하기 위해 DApp 로직을 설정하는 방법에 대한 자세한 정보는 문서의 [[USC/DApp-빌더-인프라/인프라-개요|DApp 개발자 인프라(dapp-builder-infrastructure)]] 섹션을 확인하세요.
