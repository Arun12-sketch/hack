# 오프체인 오라클 워커

> **경고:** 이 섹션에서 제공되는 모든 정보와 코드 스니펫은 교육 목적으로만 제공되며 프로덕션에 직접 배포해서는 안 됩니다.

## 개요

오프체인 오라클 워커는 크로스체인 데이터 전송 프로세스를 자동화합니다. 일반적으로 이 프로세스는 두 개의 트랜잭션이 필요합니다: 사용자가 소스 체인에서 첫 번째 트랜잭션을 시작하고, 워커가 증명을 생성하고 USC 컨트랙트를 호출하여 실행 체인에서 두 번째 트랜잭션을 처리합니다.

## 주요 이점

### 1. 사용자 경험 개선

사용자는 초기 트랜잭션에만 서명하면 됩니다. 그 외의 모든 것은 백그라운드에서 자동으로 처리됩니다.

### 2. 복잡성 감소

워커는 최종 사용자에게 부담이 될 수 있는 증명 생성, API 호출, 오류 처리를 관리합니다. 여기에는 다음이 포함됩니다:
- 증명 생성 API 서버를 통한 증명 생성
- 증명 대기
- 재시도 처리
- 복잡한 증명 구조의 적절한 포맷팅

### 3. 신뢰성 향상

자동화된 시스템은 강력한 재시도 로직을 구현하고, API 실패를 적절하게 처리하며, 적절한 오류 처리를 보장합니다.

## 워커 프로세스 흐름

워커는 다음과 같은 순차적 단계를 따릅니다:

1. **소스 체인 모니터링** - 이벤트 감지
2. **블록 증명 대기** - 증명 완료 확인
3. **증명 생성** - 머클 증명 및 연속성 증명 생성
4. **USC 컨트랙트 호출** - 증명 데이터와 함께 호출
5. **실행 결과 처리** - 성공/실패 처리

## 구현 우선순위

개발자는 다음을 통해 견고성을 우선시해야 합니다:

- **이벤트 추적** - 이벤트 로깅 및 지속성
- **캐치업 메커니즘** - 실패 후 누락된 이벤트 복구
- **리플레이 방지** - 이벤트 추적을 통한 중복 실행 방지
- **다중 노드 모니터링** - 여러 노드 감시
- **포괄적 재시도 로직** - 실패한 작업에 대한 재시도
