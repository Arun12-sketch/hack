# 인프라 개요

> **경고:** 이 섹션에서 제공되는 모든 정보와 코드 스니펫은 교육 목적으로만 제공되며 프로덕션에 직접 배포해서는 안 됩니다.

## 1. 소스 체인 스마트 컨트랙트

- **배포:** Ethereum, Sepolia
- **목적:** DApp 로직을 지원하고 검증 가능한 이벤트를 발생시킴
- **요구사항:** 추출 가능한 필드가 포함된 이벤트 발생; 소스 체인별 핸들링
- **예시:** 토큰 브릿지 사용을 위해 `TokensBurned` 이벤트를 발생시키는 ERC20 컨트랙트

## 2. 유니버설 스마트 컨트랙트 (USC)

- **배포:** Creditcoin
- **핵심 기능:** 네이티브 쿼리 검증자 프리컴파일(주소 `0x0FD2`)을 통해 크로스체인 데이터 검증
- **책임:**
  - 증명 및 트랜잭션 데이터 수신
  - 동기적 검증
  - 검증된 데이터 추출 및 처리
  - 비즈니스 로직 실행 또는 위임
- **예시:** 소스 체인의 소각 이벤트 데이터를 해석하는 토큰 브릿지 USC

## 3. DApp 비즈니스 로직 스마트 컨트랙트

- **배포:** Creditcoin
- **구성 옵션:**
  - 결합 패턴 (USC + 로직을 하나의 컨트랙트에)
  - 분리 패턴 (모듈식, 복잡한 DApp에 권장)
- **책임:**
  - 상태 관리
  - 비즈니스 로직 구현
  - 접근 제어 (일반적으로 USC 전용)
- **통합:** USC가 특별 권한을 받고 검증 후 비즈니스 로직 호출

## 4. 오라클 워커

- **배포:** 오프체인 서비스
- **워크플로우:** 이벤트 감지 → 증명 대기 → 증명 생성 → USC 호출 → 검증/실행
- **책임:**
  - 이벤트 모니터링
  - 증명 추적
  - 증명 가져오기
  - 트랜잭션 제출 및 재시도 로직

## 완전한 크로스체인 흐름

사용자 트랜잭션 → 오라클 워커 감지 → USC 검증 → 비즈니스 로직 실행 → 상태 업데이트
