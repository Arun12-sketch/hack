# 소스 체인 스마트 컨트랙트

> **주의:** 이 섹션에서 제공되는 모든 정보와 코드 스니펫은 교육 목적으로만 제공되며, 프로덕션 환경에 직접 배포하기 위한 것이 아닙니다.

## 소스 체인 스마트 컨트랙트란?

소스 체인 스마트 컨트랙트는 Creditcoin Oracle이 지원하는 이더리움과 같은 클라이언트 체인에 존재합니다. 이러한 컨트랙트는 두 가지 책임을 수행합니다: Universal DApp 로직을 지원하고, 필요한 검증 데이터와 함께 이벤트를 발생시킵니다.

### 소스 체인 로직

Universal DApp의 대부분의 로직과 데이터는 소스 체인이 아닌 Creditcoin(실행 체인)의 컨트랙트에 있어야 합니다.

모범 사례:
- 가스 비용과 복잡성을 줄이기 위해 소스 체인 로직을 최소화하세요
- 오라클의 검증 기능을 활용할 수 있는 Creditcoin에 비즈니스 로직을 유지하세요
- 소스 체인 컨트랙트는 주로 크로스 체인 작업을 트리거하는 이벤트 발생용으로 사용하세요

### 이벤트 발생

이것은 *소스 체인*이 *실행 체인*과 통신하는 방법입니다. 소스 체인 컨트랙트가 이벤트를 발생시키면, 해당 이벤트는 트랜잭션의 영수증 로그의 일부가 되며, Creditcoin에서 암호학적으로 검증할 수 있습니다.

이더리움에서 ERC20 토큰을 소각하고 Creditcoin에서 해당하는 ERC20 토큰을 발행하는 간단한 DApp을 원한다고 상상해 보세요.

이벤트 설계 고려사항:
- 효율적인 필터링을 위해 `indexed` 매개변수를 사용하세요 (최대 3개의 indexed 매개변수)
- DApp에 필요한 모든 데이터를 이벤트 매개변수에 포함하세요
- USC 컨트랙트에서의 파싱을 단순화하기 위해 이벤트 시그니처를 일관되게 유지하세요

## 예제 소스 체인 컨트랙트

다음 예제는 토큰 브릿지 DApp을 지원하는 간단한 ERC20 컨트랙트를 보여줍니다. 토큰이 '소각'(소각 주소로 전송)되면, 커스텀 `TokensBurned` 이벤트가 발생하며, 이는 Creditcoin에서 검증되어 토큰 발행을 트리거할 수 있습니다.

주요 특징:
- 오프체인 워커의 쉬운 필터링을 위한 커스텀 `TokensBurned` 이벤트를 발생시킵니다
- 토큰 소각을 나타내기 위해 소각 주소(`0x...01`)를 사용합니다
- `TokensBurned` 이벤트는 모든 필요한 데이터(`from`, `value`)를 포함합니다
- 워커는 이 특정 이벤트 시그니처를 쉽게 필터링한 다음 USC 컨트랙트에 제출할 증명을 생성할 수 있습니다

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract TestERC20 is ERC20 {
    address public constant BURN_ADDRESS = address(1); // 0x...01

    /// @notice Emitted when tokens are burned (sent to the burn address).
    /// @param from The address burning their tokens
    /// @param value The amount of tokens burned
    event TokensBurned(address indexed from, uint256 value);

    constructor() ERC20("Burn Test", "TEST") {
        // Mint sender initial supply
        _mint(msg.sender, 1_000_000 ether);
    }

    /// @notice "Burn" by transferring tokens to the 0x...01 sink address.
    /// @dev This does NOT reduce totalSupply; it only makes tokens inaccessible.
    /// @param amount The amount of tokens to burn
    /// @return success Whether the transfer succeeded
    function burn(uint256 amount) external returns (bool) {
        _transfer(msg.sender, BURN_ADDRESS, amount);
        emit TokensBurned(msg.sender, amount);
        return true;
    }
}
```
