# 소스 체인 스마트 컨트랙트

> **오래된 문서**: 이전 버전의 USC 테스트넷 문서를 보고 계십니다. 최신 문서는 [USC 문서](https://docs.creditcoin.org/usc)를 방문해 주세요.

## 소스 체인 스마트 컨트랙트란?

소스 체인 스마트 컨트랙트는 이더리움과 같은 체인에서 작동하며 Creditcoin 오라클의 지원을 받습니다. 이러한 컨트랙트는 두 가지 주요 기능을 처리합니다:

1. Universal DApp에 필요한 모든 소스 체인 로직을 지원합니다.
2. Creditcoin 오라클 쿼리로 변환될 수 있는 이벤트를 발생시킵니다.

### 소스 체인 로직

Universal DApp의 대부분의 로직과 데이터는 소스 체인이 아닌 실행 체인의 DApp 컨트랙트에 존재해야 합니다. 자산 이전과 같은 최소한의 작업은 여전히 소스 체인 실행이 필요할 수 있지만, 개발자는 이 범위를 최소화해야 합니다. Universal DApp은 비즈니스 로직이 주로 Creditcoin의 실행 체인에 있는 것에서 강점을 얻기 때문입니다.

### 이벤트 발생

이 메커니즘은 소스 체인과 실행 체인 간의 통신을 가능하게 합니다. 예를 들어, 토큰 브릿지 DApp은 이더리움에서 ERC20 토큰을 소각하고 Creditcoin에서 해당 토큰을 민팅하며, 신호 전달을 위해 `TokensBurnedForBridging`과 같은 이벤트를 사용합니다.

### 예시 소스 체인 컨트랙트

이것은 Sepolia에서 ERC20 토큰을 소각하면서 Creditcoin의 Universal DApp 컨트랙트에서 해당 토큰을 민팅하는 크로스 체인 토큰 브릿지를 보여줍니다.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract TestERC20 is ERC20 {
    address public constant BURN_ADDRESS = address(1); // 0x...01

  /// This event is bridged by the Creditcoin Oracle to trigger token minting
  /// @notice Emitted when tokens are burned (sent to the burn address).
  /// @param from The address burning their tokens
  /// @param value The amount of tokens burned
  event TokensBurnedForBridging(address indexed from, uint256 indexed value);

  constructor(string memory name_, string memory symbol_) ERC20(name_, symbol_) {
      _mint(msg.sender, 1_000_000 ether);
  }

  /// The `TokensBurnedForBridging` event emitted by this call is bridged via the
  /// Creditcoin Oracle to mint tokens on Creditcoin
  /// @notice "Burn" by transferring tokens to the 0x...01 sink address.
  function burn(uint256 amount) external returns (bool) {
      _transfer(msg.sender, BURN_ADDRESS, amount);
      emit TokensBurnedForBridging(msg.sender, amount);
      return true;
  }
}
```
