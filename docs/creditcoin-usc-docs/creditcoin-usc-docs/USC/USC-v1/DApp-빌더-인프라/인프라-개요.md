# 인프라 개요

> **참고**: 이 문서는 이전 USC 테스트넷 버전에 대한 오래된 문서입니다. [[USC/소개|현재 USC 문서]]를 참조하시기 바랍니다.

## 개요

DApp 빌더로서 인프라의 핵심 구성 요소를 이해하는 것이 중요합니다. 이 섹션에서는 USC 기반 DApp을 구성하는 주요 요소들을 설명합니다.

## 인프라 구성 요소

### 1. 소스 체인 스마트 컨트랙트

- Ethereum과 같은 네트워크에 배포됩니다
- DApp 특화 로직을 지원하고 오라클 쿼리로 변환 가능한 이벤트를 발생시킵니다
- 예시: 토큰 브릿지를 위해 전송 이벤트를 발생시키는 ERC20 컨트랙트

### 2. 유니버설 스마트 컨트랙트 (USC)

- Creditcoin 실행 체인에 위치합니다
- 오라클 결과를 해석하고 요청을 DApp 비즈니스 로직 컨트랙트로 전달합니다
- 사용 사례에 특화된 오라클 제공 데이터를 처리합니다

### 3. DApp 비즈니스 로직 스마트 컨트랙트

- Creditcoin에 상주합니다
- DApp의 상태와 로직이 존재하는 곳입니다
- USC는 오라클 결과에 따라 작업을 트리거할 수 있는 관리 권한을 부여받을 수 있습니다

### 4. 오라클 워커

- 오프체인에서 운영됩니다
- 소스 체인 컨트랙트 이벤트를 모니터링합니다
- 오라클 쿼리를 자동으로 제출하고 처리 완료 시점을 감지합니다
- 오라클 결과를 활용하기 위한 USC 호출을 시작하여, 사용자 트랜잭션 서명을 3회에서 1회로 줄입니다

## 주요 이점

이 아키텍처는 소스 체인의 트랜잭션이 Creditcoin의 DApp 로직을 자동으로 트리거할 수 있게 하여, 원활한 상호운용성을 제공합니다.
