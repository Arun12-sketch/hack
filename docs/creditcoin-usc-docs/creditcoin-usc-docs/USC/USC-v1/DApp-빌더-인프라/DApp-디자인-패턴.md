# DApp 디자인 패턴

> [!danger] 오래된 문서
> 이 문서는 이전 버전의 USC 테스트넷에 대한 문서입니다. 최신 문서는 [USC 문서](https://docs.creditcoin.org/usc)를 참조하세요.

크로스체인 DApp은 최대한 유연하게 설계된 Universal Smart Contract를 사용합니다. 이를 통해 이더리움과 같은 소스 체인의 데이터를 Creditcoin Oracle을 통해 실행 체인으로 안전하게 크로스체인 전송할 수 있습니다.

그러나 대부분의 프로젝트는 특정 패턴을 따르는 것이 가장 좋습니다.

## Creditcoin Oracle 디자인 패턴

### 소스 체인 데이터 흐름 패턴

**소스 체인의 로직을 최소한으로 유지하세요!**

1. 사용자가 소스 체인 스마트 컨트랙트를 호출합니다
2. 소스 체인 컨트랙트가 하나 이상의 이벤트를 발행합니다

### 실행 체인 데이터 흐름 패턴

**Universal Smart Contract 실행을 최대한 원활하게 만드세요!**

1. 오프체인 워커가 소스 체인 스마트 컨트랙트의 이벤트를 수신합니다
2. 오프체인 워커가 해당 이벤트에 대한 오라클 쿼리를 생성합니다
3. 오라클 쿼리가 처리되고 데이터가 Creditcoin에서 사용 가능해집니다
4. Universal DApp이 처리된 쿼리 데이터를 사용하여 비즈니스 로직을 트리거합니다

## 모범 사례

1. Universal DApp은 Creditcoin Oracle에 관련된 모든 이벤트를 발행하는 단일 소스 체인 컨트랙트를 가져야 합니다
2. 이벤트는 모호하지 않아야 합니다. 각 오라클 쿼리 유형에 대해 고유한 이벤트를 사용하세요
3. 이벤트 이름은 크로스체인 기능 시작을 나타내도록 명확하게 지정해야 합니다
4. `transfer`와 같은 일반적인 이벤트를 사용하지 마세요. 대신 브리징 전용 이벤트로 래핑하세요
5. 발행되는 이벤트에 모든 관련 크로스체인 정보를 추가하세요
