# USC 아키텍처 개요

## 경고 안내
**오래된 문서:** 이 문서는 이전 USC 테스트넷 버전을 다루고 있습니다. 최신 문서는 다음에서 확인할 수 있습니다: [[USC/소개|usc documentation]]

## 개요
탈중앙화 오라클은 중앙화된 신뢰에 의존하지 않고 외부 블록체인 데이터를 제공합니다. 대신 독립적인 행위자들이 데이터 제공의 각 단계를 실행하여 단일 주체가 결과를 통제하는 것을 방지합니다. Creditcoin의 탈중앙화 오라클은 Universal Smart Contracts(USC)가 모든 블록체인의 상태에 접근할 수 있도록 하여 유동성 풀 통합과 새로운 크로스체인 로직을 가능하게 합니다.

## 주요 용어

- **소스 체인:** Creditcoin 브리징을 지원하는 클라이언트 체인 (Ethereum과 같은 EVM 체인부터 시작)
- **실행 체인:** 오라클 인프라와 USC 지원을 호스팅하는 Creditcoin 블록체인
- **증명(Attestation):** 합의를 통해 검증된 소스 체인 블록 데이터에 대한 암호화 서약
- **증명(Proof):** 트랜잭션 구성 요소가 소스 체인에서 발생했음을 검증하는 암호화 검증
- **쿼리:** 소스 체인, 블록 번호, 트랜잭션 세부 정보를 지정하여 특정 트랜잭션 필드를 증명하기 위한 요청

*참고: 전체 트랜잭션은 크기가 크고 STARK와 같은 증명을 통해 처리하기에 비용이 많이 들 수 있으므로 트랜잭션 부분만 증명됩니다.*

## 아키텍처 개요

Creditcoin 탈중앙화 오라클은 세 가지 탈중앙화 행위자 그룹으로 구성됩니다:

### 증명자(Attestors)
소스 체인 상태 변경을 보고합니다. Creditcoin은 개별 보고를 신뢰하는 대신 여러 증명자 간의 합의를 요구합니다.

### 증명 생성자(Provers)
세 단계를 통해 트랜잭션 구성 요소에 대한 사용자 쿼리에 응답하는 암호화 증명을 생성합니다:
1. 증명(attestation)을 사용하여 블록이 소스 체인 상태에 포함되어 있음을 증명
2. 머클 증명을 사용하여 해당 블록에 트랜잭션이 존재함을 검증
3. 두 속성을 모두 검증하는 암호화 증명 생성

### 검증자(Validators)
Creditcoin의 권한 집합 구성원으로서 증명자 서명과 증명 유효성을 검증하여, 증명을 포함한 블록을 확정하기 전에 올바른 작업이 실행되었는지 확인합니다.

### 상호운용성 영향
서드파티 개발자는 "다른 체인의 데이터에 대한 안전하고 **원활한** 접근"을 통해 Creditcoin 컨트랙트를 구축할 수 있으며, 중앙화된 중개자 없이 신뢰가 필요 없는 크로스체인 자산 관리가 가능합니다.

## 데이터 제공 흐름

시스템은 순차적으로 작동합니다: 증명자가 소스 체인 블록을 모니터링 → DApp이 쿼리 제출 → 저장된 증명이 대상 트랜잭션을 검증 → 증명 결과가 온체인에 저장 → USC가 읽고 비즈니스 로직을 실행합니다.

## 크로스체인 대출 사용 사례 예시

이 아키텍처는 크로스체인 대출 인프라를 통해 적용 사례를 보여줍니다:
- 진한 녹색 화살표는 증명(attestation) 프로세스를 나타냄
- 연한 녹색 화살표는 STARK 증명을 생성하는 증명(proving) 프로세스를 나타냄
- 빨간색과 파란색 화살표는 동일한 프로세스의 대체 시각화를 나타냄
