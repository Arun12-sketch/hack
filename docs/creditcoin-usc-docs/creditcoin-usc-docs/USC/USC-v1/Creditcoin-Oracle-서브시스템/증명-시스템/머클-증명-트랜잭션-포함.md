# 머클 증명 트랜잭션 포함

## 개요

이 문서는 머클 증명이 블록체인 블록 내 트랜잭션 포함을 어떻게 효율적으로 검증하는지 설명합니다. 특히 Creditcoin 분산 오라클에서 사용되는 STARK 증명 환경에서의 활용을 다룹니다.

## 핵심 효율성 이점

주요 장점은 계산 효율성입니다. 블록 내 모든 트랜잭션을 순차적으로 검사하는 대신, 머클 증명은 "n개의 트랜잭션이 있는 블록에서 log₂(n)개의 해시"만 접근하면 됩니다. 이는 백만 개의 트랜잭션에 대해 약 20개의 해시만 필요함을 의미합니다. STARK 환경에서 이 접근 방식은 검증 시간을 약 30분에서 5분으로 단축시킵니다.

## 핵심 개념 설명

**머클 트리**: 모든 요소를 검사하지 않고도 대규모 데이터셋의 무결성을 빠르게 검증할 수 있는 균형 잡힌 해시 기반 데이터 구조입니다.

**머클 루트**: 트리 정점에 있는 단일 암호화 해시로, 포함된 모든 데이터의 포괄적 검증을 가능하게 합니다.

**필드**: 오라클이 계산 자원과 저장 공간을 절약하기 위해 선택적으로 제공하는 개별 트랜잭션 구성 요소(상태나 전송 값 등)입니다.

**Starknet Pederson 머클 트리**: 증명자가 정직한 머클 검증을 인증하는 암호화 증명을 생성할 수 있게 해주는 STARK 최적화 머클 트리 변형입니다.

## 증명 프로세스

시스템은 원하는 트랜잭션 필드를 패키징하고, 다른 트랜잭션들과 함께 해시한 다음, 루트 해시가 형성될 때까지 머클 노드를 재귀적으로 결합합니다. 대상 트랜잭션으로 가는 경로의 형제 노드 해시들과 머클 루트를 결합하여, 증명자는 검증을 위해 STARK 증명으로 래핑된 머클 증명을 생성합니다.
