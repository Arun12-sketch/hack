# 증명과 연속성 증명

> **상태 알림:** 이 문서는 구버전 USC 테스트넷을 다룹니다. 최신 문서는 [[USC/소개|USC 문서]]에서 확인할 수 있습니다.

## 개요

연속성 증명(Continuity proofs)은 Creditcoin 분산 오라클이 블록체인 간에 데이터를 안전하게 전송할 수 있게 하는 핵심 구성 요소입니다. 증명이 직접 연결되지 않는 곳에서 간격을 연결하여 소스 체인 블록의 끊김 없는 체인을 유지합니다.

시스템은 비용 제약으로 인해 블록별 증명 대신 간격 기반 저장 방식을 사용합니다. "모든 소스 체인 블록에 대한 증명을 Creditcoin 체인에 저장하는 것은 비용이 너무 많이 들 것입니다—특히 Solana처럼 블록을 자주 생성하는 체인의 경우에는 더욱 그렇습니다." 대신, 증명은 일정 간격(예: 10 또는 100 블록마다)으로 발생하며, 연속성 증명이 다이제스트 체인을 통해 중간 간격을 채웁니다.

## 주요 용어

**해시(Hash):** "암호화 해시는 임의의 크기의 입력을 받아 해시 값이라고 하는 고정 크기의 출력을 생성하는 결정론적 수학 함수입니다."

**머클 트리(Merkle Tree):** "머클 트리는 대규모 데이터 세트의 무결성을 효율적이고 안전하게 검증할 수 있게 하는 암호화 해시의 균형 이진 트리입니다."

**루트/머클 루트(Root/Merkle Root):** "머클 루트는 머클 트리 최상단에 있는 단일 암호화 해시입니다. 그 아래의 모든 데이터를 고유하게 요약합니다."

**다이제스트(Digest):** "Creditcoin 오라클의 맥락에서, 다이제스트는 일반적으로 블록이나 증명을 고유하게 식별하는 해시 출력을 설명합니다."

**이전 다이제스트(Previous Digest):** 바로 이전 블록의 다이제스트로, 후속 블록 다이제스트 생성에 사용됩니다.

## 블록 체이닝 메커니즘

시스템은 블록체인의 근본적인 속성을 활용합니다: 각 블록의 다이제스트는 그 내용과 이전 블록의 다이제스트를 모두 포함합니다. 이는 어떤 블록을 수정하면 모든 후속 다이제스트가 무효화되는 불변 체인을 생성합니다.

## 증명 생성 프로세스

### 1단계: 선택

증명자(Attestors)는 다음 공식을 사용하여 대상 블록을 계산합니다:

```
next_attestation_block = latest_attestation_block + attestation_interval
```

### 2단계: 가져오기

증명자는 RPC 노드에서 블록을 가져옵니다. 직접 모니터링하든 외부 엔드포인트를 쿼리하든, 증명자 네트워크 전반에 다양성이 존재한다면 상관없습니다.

### 3단계: 프래그먼트 구성

다이제스트는 순차적으로 계산되며, 각각은 이전 다이제스트를 포함합니다:

```rust
let digest = Self::hash_payload(&block_number.into(), &root, &prev_digest);
```

이는 중간 블록을 통해 증명을 연결하는 검증 가능한 체인을 생성합니다.

## 연속성 증명 검증

증명자(Provers)는 쿼리된 블록을 근처의 증명에 연결하는 증명 체인을 다음과 같이 설정합니다:

- 간격 경계 식별
- 중간 블록 가져오기
- 다이제스트 체인 구성
- 암호화 증명 생성
- Creditcoin 검증자에게 제출

검증자는 최종 다이제스트가 저장된 증명 다이제스트와 일치하는지 확인합니다. 어떤 블록이라도 수정되면 후속 다이제스트에 연쇄적으로 영향을 미쳐 불일치가 발생하고 잘못된 증명은 거부됩니다.
