# 증명 (Proving)

> **주의: 오래된 문서**
> 이 문서는 이전 버전의 USC 테스트넷에 대한 문서입니다. 최신 문서는 다음을 방문하세요: [[USC/소개|usc documentation]]

Creditcoin 오라클은 분산화된 증명자(prover) 네트워크를 통해 소스 체인 데이터를 Creditcoin 체인에 제공합니다. 각 증명자 운영자는 Creditcoin에 자체 증명자 스마트 컨트랙트를 등록하고 자체 오프체인 증명자 워커를 운영합니다. 증명을 시작하려면 사용 가능한 증명자 중 하나에 증명자 스마트 컨트랙트 호출을 통해 쿼리를 제출합니다. 그러면 쿼리가 증명자 컨트랙트에 저장되고 처리를 기다립니다. 증명자 오프체인 워커는 증명자 컨트랙트에서 새로운 쿼리를 폴링하고 수신된 순서대로 쿼리를 처리하기 시작합니다.

쿼리를 처리하기 위해 증명자는 두 가지 추가 정보 소스가 필요합니다:

1. **증명(Attestations)** - 증명자는 Creditcoin 증명 팔레트(attestation pallet)에 온체인으로 저장된 증명을 요청합니다. 이는 쿼리된 블록이 실제로 확정된 소스 체인의 일부인지 확인하는 연속성 증명(continuity proof)을 생성하는 데 필요합니다.

2. **소스 체인 블록** - 증명자는 소스 체인에 연결된 RPC 노드에서 소스 체인 블록을 요청합니다. 증명자는 이러한 블록을 신뢰할 필요가 없는데, 증명자가 증명과 연속성 증명이라는 프로세스를 사용하여 블록이 진짜인지 확인할 수 있기 때문입니다. 쿼리 트랜잭션이 포함된 소스 체인 블록은 주장된 트랜잭션이 실제로 발생했는지 확인하는 머클 증명(Merkle proof)을 구성하는 데 사용됩니다.

증명자는 연속성 증명과 머클 증명을 모두 요약하는 암호화 증명을 생성합니다. 이 증명은 증명자 컨트랙트에 제출되며, Creditcoin의 검증자 프리컴파일(verifier precompile)을 통해 온체인에서 검증됩니다. 검증이 완료되면 결과는 증명자 컨트랙트에 저장되며, 사용자 및 빌더 스마트 컨트랙트가 검증된 소스 체인 데이터로 접근할 수 있습니다.
