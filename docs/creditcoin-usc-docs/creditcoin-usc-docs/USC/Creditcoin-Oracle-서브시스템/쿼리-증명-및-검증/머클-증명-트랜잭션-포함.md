# 머클 증명 트랜잭션 포함

머클 증명에 대한 자세한 내용은 [이 글](https://medium.com/@swastika0015/merkle-proofs-explained-208a72971a50)을 참고하세요.

## 동기

트랜잭션이 특정 블록에 속하는지 어떻게 확인할 수 있을까요? 한 가지 방법은 블록의 모든 트랜잭션을 살펴보고 해당 트랜잭션을 찾는 것입니다. 하지만 이는 모든 `n`개의 트랜잭션에 접근해야 하므로 매우 비효율적입니다.

머클 증명은 `log₂(n)`개의 해시에만 접근하면 되므로 훨씬 효율적입니다. 예를 들어, 1,000,000개의 트랜잭션이 있는 블록의 경우 약 20개의 해시만 확인하면 됩니다.

## 주요 용어

- **머클 트리**: 대규모 데이터 집합의 무결성을 효율적으로 검증하는 데 사용되는 균형 잡힌 해시 트리 데이터 구조입니다.

- **루트 (머클 루트)**: 트리 최상단에 있는 단일 암호화 해시로, 해당 트리 구조 내에 저장된 모든 데이터를 빠르게 검증할 수 있게 해줍니다.

- **필드**: 블록체인 트랜잭션의 일부입니다. 예를 들어, 필드는 트랜잭션 `status`(성공/실패)이거나 전송 이벤트의 `value` 필드일 수 있습니다.

- **표준 머클 트리**: 시스템은 빠르고 효율적인 검증을 위해 네이티브 프리컴파일 검증과 함께 Keccak-256 해싱을 사용합니다.

## 머클 트랜잭션 필드 증명

트랜잭션 필드(예: ERC20 전송의 `from`, `to`, `value`)는 함께 패킹되어 트리 구조에서 리프 노드를 형성하는 해시가 됩니다. 이러한 리프들은 단일 루트 해시가 생성될 때까지 재귀적 해싱을 통해 쌍으로 결합됩니다.

대상 트랜잭션으로 가는 경로를 따라 형제 노드 해시와 머클 루트를 사용하여, 해당 트랜잭션이 특정 블록에 포함되어 있음을 암호학적으로 증명할 수 있습니다.
