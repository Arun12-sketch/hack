# 증명 (Attestation)

## 소개

증명(Attestation)은 Creditcoin 탈중앙화 오라클이 외부 소스 체인의 최종 확정된 상태를 추적하는 프로세스입니다. 각 소스 체인의 상태 전환은 탈중앙화된 증명자(attestor) 네트워크에 의해 모니터링됩니다. 각 적격 증명자는 소스 체인의 새로운 블록에 대한 자신의 관점을 나타내는 암호학적 약정(cryptographic commitment)인 증명을 생성하고, BLS 서명으로 서명합니다. 단일 증명자를 완전히 신뢰할 수 없으므로, 독립적인 증명자들 간의 합의가 필요하며, 이는 Creditcoin에 제출하기 전에 오프체인에서 투표와 서명을 집계하는 P2P 가십 네트워크를 통해 달성됩니다.

## 증명 프로세스

증명 프로세스는 다음과 같이 진행됩니다:

1. 증명자들은 새로운 최종 확정 블록을 찾기 위해 외부 소스 체인을 지속적으로 모니터링합니다.

2. 각 증명자는 양쪽 체인에 대한 자신의 관점을 기반으로, Creditcoin의 마지막 최종 확정 증명부터 현재 높이까지 소스 체인의 진행 상황을 요약합니다. 해당 소스 체인 블록 높이에 대한 모든 적격 증명자는 각자의 증명에 BLS 서명을 하고 P2P 가십 네트워크에 제출합니다.

3. 오프체인 P2P 가십 네트워크는 증명 투표를 조정하고, 검증하며, BLS 서명들을 집계합니다.

4. 정족수(quorum)에 도달하면, [VRF](https://en.wikipedia.org/wiki/Verifiable_random_function)를 사용하여 무작위 증명자 집합이 선택되어 집계된 증명을 Creditcoin에 제출합니다.

5. 그런 다음 Creditcoin 검증자들은 집계된 증명(증명자 적격성 및 집계된 BLS 서명 포함)을 검증한 후 온체인에 저장합니다.

이 접근 방식은 여러 증명자 투표를 단일 집계 서명이 포함된 하나의 트랜잭션으로 통합하여 온체인 트래픽을 줄이고, 더 큰 증명자 네트워크로 효율적으로 확장할 수 있게 합니다.
