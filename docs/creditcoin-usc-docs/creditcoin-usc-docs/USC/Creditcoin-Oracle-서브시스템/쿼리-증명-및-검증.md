# 쿼리, 증명 및 검증

## **개요**

쿼리-증명-검증 프로세스는 유니버설 스마트 컨트랙트가 외부 소스 체인의 데이터를 신뢰 없이 검증하고 사용할 수 있게 해줍니다. 이 프로세스는 네 가지 주요 단계로 구성됩니다:

1. **쿼리 단계**: 검증할 대상 트랜잭션 식별
2. **증명 생성 단계**: 머클 증명과 연속성 증명 생성
3. **검증 단계**: 증명의 암호화 검증
4. **데이터 추출 단계**: 검증된 바이트에서 트랜잭션 데이터 추출

각 단계는 이전 단계를 기반으로 구축되어, 소스 체인 트랜잭션에서 Creditcoin 스마트 컨트랙트 실행까지 신뢰 없는 검증 체인을 형성합니다.

## **증명 유형**

소스 체인에서 트랜잭션이 발생했음을 증명하기 위해 시스템은 두 가지 상호 보완적인 암호화 증명을 사용합니다:

* **머클 증명**: 특정 트랜잭션이 소스 체인의 특정 블록에 포함되어 있음을 증명합니다
* **연속성 증명**: 트랜잭션을 포함하는 블록이 최종화된 소스 체인의 일부임을 증명합니다 (증명서에 연결됨)

이 두 증명을 함께 사용하면 트랜잭션이 실제로 소스 체인에서 발생했다는 암호화적 확신을 제공하여, 신뢰 없는 크로스체인 애플리케이션을 가능하게 합니다.

이 프로세스는 다음을 활용합니다:

* **증명 생성 API 서버** (오프체인): 온디맨드로 머클 증명과 연속성 증명을 생성합니다
* [[USC/개요/USC-아키텍처-개요|Native Query Verifier Precompile]] (온체인): 증명을 동기적으로 검증하고 데이터를 추출합니다

증명 생성 API는 Creditcoin에서 인덱싱된 증명서 및 체크포인트 데이터를 쿼리하고 RPC 노드에서 소스 체인 블록을 가져온 다음, 인덱싱된 데이터에서 계산된 루트와 다이제스트를 사용하여 연속성 증명을 구성합니다. 체크포인트 사이의 쿼리의 경우, API는 체크포인트 간격 범위 내의 모든 증명서를 쿼리합니다. 이러한 증명은 USC 컨트랙트를 통해 Creditcoin에 제출되며, 여기서 프리컴파일이 동기적으로 검증합니다. 검증이 완료되면 트랜잭션 데이터를 즉시 사용할 수 있으며, 중간 저장이나 비동기 처리가 필요하지 않습니다.
