## 1. DEX — Uniswap V3 Fork

### 왜 Uniswap V3인가?

> DEX는 DeFi의 가장 기초적인 인프라입니다. 토큰 교환 없이는 어떤 DeFi 활동도 불가능합니다.
>
> 우리는 Uniswap V3를 포크했습니다. V3의 **집중화 유동성(Concentrated Liquidity)** 모델은 LP가 특정 가격 범위에만 유동성을 집중시킬 수 있어서, V2 대비 최대 4000배 자본 효율성을 제공합니다. 이는 테스트넷처럼 유동성이 제한된 환경에서 특히 중요합니다.
>
> Snowball의 모든 프로토콜 — CDP의 sbUSD, Lending의 담보 토큰, Yield의 기초 자산 — 이 모든 것이 이 DEX 위에서 거래됩니다.

### 1-1. Swap 페이지 (`/swap`)

#### 화면에 보여줄 것
- 토큰 선택 드롭다운 (From: wCTC → To: sbUSD)
- 금액 입력 → 실시간 견적 표시
- Max 버튼으로 전액 입력
- 슬리피지 설정 (기본 0.5%)
- 가격 차트 (24시간 추이)

#### 대본

> Swap 페이지입니다. 기본적인 토큰 교환 인터페이스입니다.
>
> (토큰 선택하며) wCTC를 sbUSD로 바꿔보겠습니다. 금액을 입력하면 실시간으로 예상 수령량이 계산됩니다. 이 견적은 온체인의 QuoterV2 컨트랙트를 직접 호출해서 가져오는 실제 가격입니다.
>
> Max 버튼으로 지갑 잔액 전체를 입력할 수도 있고, 슬리피지 허용치도 조절 가능합니다.
>
> (스왑 실행) Approve → Swap 두 단계 트랜잭션으로 진행됩니다. 트랜잭션 파이프라인 모달에서 각 단계의 진행 상황을 실시간으로 확인할 수 있습니다.

### 1-2. Pool 목록 페이지 (`/pool`)

#### 화면에 보여줄 것
- 프로토콜 통계 (TVL, 24h Volume, 24h Fees, Total Pools)
- 트렌딩 풀 캐러셀
- 풀 테이블 (Pair, Fee Tier, TVL, 24h Volume, APR)

#### 대본

> Pool 페이지에서는 현재 활성화된 유동성 풀 전체를 볼 수 있습니다.
>
> 상단에 프로토콜 전체 TVL, 24시간 거래량, 수수료 수익이 표시됩니다. 트렌딩 풀 섹션에서는 최근 거래량이 가장 많은 풀들을 빠르게 확인할 수 있습니다.
>
> 각 풀마다 수수료 티어, TVL, APR이 표시되어서 LP들이 어디에 유동성을 공급할지 비교할 수 있습니다.

### 1-3. Pool 상세 + LP 포지션 생성 (`/pool/[pair]`)

#### 화면에 보여줄 것
- **유동성 히스토그램** — 틱별 유동성 분포 시각화
- **가격 범위 프리셋** (Narrow / Common / Wide / Full / Custom)
- 드래그 가능한 슬라이더로 커스텀 범위 설정
- 입금 패널 (HALF/MAX 버튼, 토큰 비율 바)
- 예상 APR 표시

#### 대본

> 특정 풀을 클릭하면 상세 페이지로 들어옵니다. 여기가 Uniswap V3의 핵심인 **집중화 유동성 포지션 생성** 화면입니다.
>
> (히스토그램 가리키며) 이 차트는 현재 풀에서 각 가격대에 얼마나 유동성이 집중되어 있는지를 보여줍니다. LP는 이걸 참고해서 자기 유동성을 어디에 배치할지 결정합니다.
>
> 프리셋으로 Narrow(좁은 범위, 고위험 고수익), Common(일반적), Wide(넓은 범위, 안정적) 등을 선택하거나, 슬라이더를 직접 드래그해서 커스텀 범위를 설정할 수 있습니다.
>
> 입금할 금액을 넣으면 현재 풀 가격 기준으로 두 토큰의 최적 비율이 자동 계산됩니다. HALF 버튼은 잔액의 절반, MAX는 전액입니다.

### 1-4. 내 포지션 관리 (`/pool/positions`)

#### 화면에 보여줄 것
- LP 포트폴리오 요약 (Total Value, Active Count, Uncollected Fees)
- 개별 포지션 카드 (상태 뱃지: In Range / Out of Range)
- 범위 시각화 바 (초록 = 범위 내, 빨강 = 범위 밖)

#### 대본

> 내 포지션 페이지에서는 현재 보유한 모든 LP 포지션을 한눈에 관리합니다.
>
> 포트폴리오 전체 가치, 활성 포지션 수, 미수령 수수료가 상단에 표시됩니다.
>
> 각 포지션 카드에서 가장 중요한 건 이 **범위 상태 표시**입니다. 초록색이면 현재 가격이 내가 설정한 범위 안에 있어서 수수료를 벌고 있다는 뜻이고, 빨간색이면 범위를 벗어나서 수익이 발생하지 않고 있다는 의미입니다.

## 2. CDP — Liquity V2 Fork

### 왜 Liquity V2인가?

> CDP(Collateralized Debt Position)는 담보를 맡기고 스테이블코인을 빌리는 프로토콜입니다.
>
> 우리는 Liquity V2를 포크했습니다. Liquity V2는 MakerDAO와 달리 **거버넌스 없이** 운영되며, 유저가 직접 이자율을 설정하는 독특한 모델을 씁니다. 이자율이 낮을수록 청산 우선순위가 높아지고, 높을수록 안전하지만 비용이 큽니다. 이 게임이론적 설계가 시장 효율적인 금리를 자연스럽게 형성합니다.
>
> 여기서 발행되는 **sbUSD**가 Snowball 생태계의 기축 스테이블코인이 됩니다. DEX에서 거래되고, Lending에서 담보나 대출 자산으로 쓰이고, Yield Vault에서 자동 복리됩니다.

### 2-1. Borrow 페이지 (`/liquity/borrow`)

#### 화면에 보여줄 것
- **브랜치 선택** (wCTC / lstCTC 탭) → 다른 담보 유형
- **시스템 통계** (담보 가격, TVL, 총 부채, TCR)
- **Open Trove 다이얼로그**:
  - 담보 입력 (HALF/MAX 버튼)
  - 대출(sbUSD) 입력 (SAFE 버튼)
  - **이자율 슬라이더** — 그라디언트 게이지 + 시장 평균 마커
  - **포지션 서머리** — CR, 청산가, 선불 수수료, 연간 비용
- **내 Trove 카드** — 활성 트로브 목록 + Edit/Delegate/Close 버튼
- **시스템 Trove 카드** — 전체 트로브 테이블 (이자율순 정렬)

#### 대본

> Borrow 페이지입니다. 여기서 담보를 맡기고 sbUSD를 빌립니다.
>
> (브랜치 탭 가리키며) Liquity V2는 **멀티 브랜치** 구조입니다. wCTC와 lstCTC 두 가지 담보를 각각 독립적인 풀에서 관리합니다. lstCTC는 Liquid Staked CTC로, 스테이킹 보상을 받으면서 동시에 담보로 활용할 수 있습니다.
>
> (Open Trove 클릭) Trove를 열어보겠습니다. 담보량을 입력하고, 빌릴 sbUSD 양을 정합니다. SAFE 버튼을 누르면 안전한 수준의 대출량을 자동 계산해줍니다.
>
> (이자율 슬라이더 가리키며) 이것이 Liquity V2의 핵심입니다. **이자율을 유저가 직접 설정**합니다. 이 그라디언트 게이지에서 왼쪽(낮은 이자율)은 빨간색 — 저렴하지만 청산 우선순위가 높아서 위험합니다. 오른쪽(높은 이자율)은 초록색 — 비용은 크지만 안전합니다. 흰색 마커가 현재 시장 평균 이자율입니다.
>
> (포지션 서머리 가리키며) 실시간으로 담보비율(CR), 청산가, 7일 선불 수수료, 연간 이자 비용이 계산됩니다. CR이 빨간색이면 MCR(최소 담보비율) 이하라서 개설 불가합니다.
>
> (Trove 개설 실행) Approve → Open Trove 트랜잭션이 실행됩니다.

#### 추가 시연: 내 Trove 관리

> (내 Trove 카드 가리키며) 개설된 Trove가 여기 표시됩니다. 각 Trove마다 담보량, 부채, 이자율, 개별 담보비율(ICR)이 보이고, 이 미니 게이지에서 내 이자율이 시장 대비 어디에 위치하는지 한눈에 알 수 있습니다.
>
> Edit 버튼으로 담보 추가/인출, 부채 조정, 이자율 변경이 가능합니다. Close 버튼으로 부채를 전액 상환하고 담보를 회수할 수 있습니다.
>
> (시스템 Trove 테이블 가리키며) 시스템 전체 Trove도 조회 가능합니다. 이자율 순으로 정렬되어 있고, ICR이 빨간색인 Trove는 청산 위험이 높다는 의미입니다.

### 2-2. Earn 페이지 (`/liquity/earn`)

#### 화면에 보여줄 것
- **Stability Pool 통계** (풀 규모, 내 예치금, 담보 수익)
- **예치(Deposit) 카드** — sbUSD 입금
- **인출/클레임 카드** — sbUSD 인출 + 담보 보상 수령

#### 대본

> Earn 페이지는 **Stability Pool**입니다. sbUSD를 예치하면 청산 과정에서 발생하는 담보 수익을 얻습니다.
>
> 누군가의 Trove가 담보비율 미달로 청산되면, Stability Pool의 sbUSD가 부채 상환에 사용되고, 그 대가로 청산된 담보(wCTC 또는 lstCTC)가 풀에 분배됩니다. 보통 담보 가치가 부채보다 크기 때문에 예치자는 차익을 얻게 됩니다.
>
> (Deposit 카드) sbUSD를 예치하고, (Claim 버튼) 쌓인 담보 보상을 수령할 수 있습니다.

## 3. Lending — Morpho Blue Fork

### 왜 Morpho Blue인가?

> Lending은 일반적인 예금/대출 프로토콜입니다. Aave나 Compound 같은 것이지만, 우리는 **Morpho Blue**를 선택했습니다.
>
> Morpho Blue는 Aave와 근본적으로 다릅니다. Aave가 모든 자산을 하나의 풀에 묶는 반면, Morpho Blue는 **독립된 마켓** 단위로 운영됩니다. 각 마켓은 고유한 담보-대출 페어, LLTV(청산 임계치), 오라클, 금리 모델을 가집니다. 이 모듈러 설계 덕분에 새로운 자산 페어를 거버넌스 없이 누구나 만들 수 있고, 리스크가 마켓 단위로 격리됩니다.
>
> 또한 Morpho Blue의 금리 모델인 **Adaptive Curve IRM**은 시장 이용률에 따라 금리를 자동 조정하여 최적의 유동성 균형을 유지합니다.

### 3-1. Supply 페이지 (`/morpho/supply`)

#### 화면에 보여줄 것
- 마켓 카드 그리드 (마켓별 Supply APY, Borrow APR, 이용률 바)
- 내 Supply 잔액 표시
- Supply/Withdraw 다이얼로그

#### 대본

> Lending의 Supply 페이지입니다. 각 카드가 하나의 독립된 마켓을 나타냅니다.
>
> (카드 가리키며) Supply APY, Borrow APR, 그리고 이용률이 표시됩니다. 이용률이 높을수록 대출 수요가 많다는 뜻이고, 공급자의 수익률도 올라갑니다.
>
> (Supply 클릭) 토큰을 공급하면 이자를 받습니다. 금리는 Adaptive Curve IRM이 이용률에 따라 자동 조정합니다.

### 3-2. Borrow 페이지 (`/morpho/borrow`)

#### 화면에 보여줄 것
- 마켓 카드 (LLTV, 대출 가능 금액, 내 포지션)
- 내 포지션: 담보, 대출 잔액, **Health Factor** (색상 코딩)
- 4개 액션 다이얼로그: 담보 공급 / 대출 / 상환 / 담보 인출

#### 대본

> Borrow 페이지에서는 담보를 넣고 대출을 받습니다.
>
> (Health Factor 가리키며) 가장 중요한 지표가 이 **Health Factor**입니다. 1 이상이면 안전, 1 미만이면 청산 위험입니다. 실시간으로 계산되어 포지션 상태를 즉시 파악할 수 있습니다.
>
> (액션 버튼들) 담보 공급, 대출, 상환, 담보 인출 — 4가지 액션을 각각 독립적으로 실행할 수 있습니다. Morpho Blue의 특징은 이 모든 것이 하나의 싱글톤 컨트랙트에서 처리된다는 점입니다.

## 4. Yield Vaults — Beefy Finance Fork

### 왜 Beefy Finance인가?

> Yield Vault는 **자동 복리** 프로토콜입니다. DeFi에서 수익을 얻으면 보통 수동으로 보상을 수확(harvest)하고 재투자해야 하는데, Yield Vault가 이걸 자동화합니다.
>
> Beefy Finance를 포크한 이유는 그 검증된 Vault 아키텍처 때문입니다. 사용자는 토큰을 예치하기만 하면, Vault가 주기적으로 수익을 수확하고 자동으로 재투자합니다. 복리 효과로 단순 스테이킹보다 높은 APY를 달성할 수 있습니다.

### 4-1. Yield Vaults 대시보드 (`/yield`)

#### 화면에 보여줄 것
- 헤더 통계 (총 예치금, 활성 볼트 수, 최고 APY)
- 볼트 카드 그리드:
  - 볼트 이름 + 설명 + 상태 뱃지 (Active/Paused)
  - TVL, 예상 APY, 내 예치금
  - Last Harvest 시간, 출금 수수료
  - Deposit/Withdraw 버튼
- Deposit/Withdraw 다이얼로그

#### 대본

> Yield Vaults 페이지입니다. 각 볼트가 특정 전략을 실행하는 자동 복리 금고입니다.
>
> (카드 가리키며) 각 볼트는 TVL, 예상 APY, 마지막 수확(Harvest) 시간을 보여줍니다. 사용자는 토큰을 예치하기만 하면 됩니다.
>
> (Deposit 실행) 예치하면 볼트 토큰(share)을 받고, 시간이 지나면서 share의 가치가 올라갑니다. 출금 시 늘어난 가치만큼 수익을 실현합니다.
>
> 이 모든 과정이 자동이라 사용자가 매일 수동으로 수확하고 재투자할 필요가 없습니다.

## 5. DN Bridge — Wormhole + Custom Worker

### 왜 브릿지가 필요한가?

> 크로스체인 브릿지는 서로 다른 블록체인 간에 자산을 이동시키는 인프라입니다.
>
> Snowball은 Creditcoin 테스트넷에서 운영되지만, 다른 체인의 자산(예: Ethereum의 USDC)을 가져와야 합니다. 이를 위해 **Wormhole 프로토콜**을 활용한 브릿지와, 자체 개발한 **오프체인 Worker**를 결합했습니다.
>
> 전체 흐름은: USDC(Creditcoin) → Wormhole → DN(Sepolia) → BridgeBurn → Worker 자동 감지 → USC Mint(Creditcoin USC Testnet) 입니다.

### 5-1. Bridge 페이지 (`/bridge`)

#### 화면에 보여줄 것
- **체인 대시보드** — 3개 체인의 잔액 실시간 표시
  - CTC Testnet: USDC
  - Eth Sepolia: DN
  - CTC USC Testnet: DN
- **브릿지 파이프라인 카드**:
  - 금액 입력
  - 6단계 진행 표시:
    1. USDC Approve
    2. Wormhole 전송
    3. DN Mint (Sepolia)
    4. Bridge Burn
    5. Attestation 대기 (~5분)
    6. USC 자동 Mint
  - 각 단계별 상태 (완료/대기/에러/타임아웃)

#### 대본

> Bridge 페이지입니다. 3개 체인에 걸친 크로스체인 자산 이동을 하나의 파이프라인으로 처리합니다.
>
> (체인 대시보드 가리키며) 상단에서 각 체인별 잔액을 실시간으로 확인할 수 있습니다.
>
> (파이프라인 가리키며) 브릿지 과정은 6단계입니다. 사용자는 금액만 입력하고 Start Bridge를 누르면, 각 단계가 순차적으로 실행됩니다. 특히 5단계 Attestation은 약 5분 정도 걸리는데, 이 과정에서 페이지를 닫았다가 다시 열어도 **세션이 localStorage에 저장**되어 있어서 이어서 진행할 수 있습니다.
>
> 마지막 6단계에서는 우리가 자체 개발한 **오프체인 Worker**가 BridgeBurn 이벤트를 자동 감지하고, 검증 후 대상 체인에서 USC를 민팅합니다. 사용자가 추가로 할 일은 없습니다.

## 6. CDP Agent — ERC-8004 (자체 설계)

### 왜 온체인 AI Agent인가?

> 지금까지 보신 DeFi 프로토콜들은 모두 사용자가 직접 조작해야 합니다. 이자율 조정, 담보비율 관리, 포지션 리밸런싱 — 이걸 24시간 감시하고 대응하는 건 현실적으로 어렵습니다.
>
> 그래서 우리는 **ERC-8004** 표준을 기반으로 온체인 AI Agent 시스템을 설계했습니다. 이건 기존 프로토콜의 포크가 아닌 **Snowball 자체 설계**입니다.
>
> Agent는 사용자의 Trove를 모니터링하고, 시장 상황에 따라 이자율을 자동 조정합니다. 핵심은 **사용자가 Agent에게 권한을 위임하되, 범위를 제한**할 수 있다는 것입니다. 최소/최대 이자율 범위, 토큰 한도, 만료 기한 등을 설정해서 Agent가 무제한으로 행동하는 것을 방지합니다.

### 6-1. Agent 마켓플레이스 (`/agent`)

#### 화면에 보여줄 것
- **My Agents** 섹션 — 내가 등록한 에이전트
- **All Agents** 섹션 — 마켓플레이스 전체 에이전트
- AgentCard (이름, 타입, 상태 뱃지)

#### 대본

> Agent 마켓플레이스입니다. 등록된 모든 AI Agent를 한눈에 볼 수 있습니다.
>
> 각 Agent 카드에는 이름, 타입, 활성 상태가 표시됩니다. 사용자는 원하는 Agent를 선택해서 자기 포지션 관리를 위임할 수 있습니다.
>
> Register 버튼으로 새 Agent를 등록할 수도 있습니다.

### 6-2. Agent 등록 (`/agent/register`)

#### 화면에 보여줄 것
- 등록 폼 (Name, Agent Type, Endpoint Address, Token URI)
- Register 트랜잭션 실행

#### 대본

> Agent 등록 화면입니다. Agent의 이름, 타입, 그리고 가장 중요한 **Endpoint Address** — 이 Agent가 실행될 스마트 컨트랙트 주소를 입력합니다.
>
> ERC-8004 표준에 따라 IdentityRegistry에 등록되면, 다른 사용자들도 이 Agent를 발견하고 사용할 수 있게 됩니다.

### 6-3. Agent 프로필 (`/agent/[id]`)

#### 화면에 보여줄 것
- **AgentProfileHeader** — 이름, 타입, 엔드포인트, 검증 뱃지
- **ReputationSection** — 평판 점수, 성공률, 리뷰
- **RunAgentButton** — 수동 실행 카드 (troveId 입력)
- **ActivityLog** — 서버 실행 기록 + 온체인 이벤트

#### 대본

> Agent 프로필 페이지입니다. Agent의 상세 정보와 실적을 확인할 수 있습니다.
>
> (평판 섹션) 성공률과 사용자 리뷰로 Agent의 신뢰도를 판단할 수 있습니다.
>
> (RunAgent 카드) 여기서 Agent를 수동으로 한 번 실행해볼 수 있습니다. troveId를 입력하면 Agent가 현재 시장 상황을 분석하고, 행동이 필요하면 트랜잭션을 실행합니다. 실행 결과에는 Agent의 **추론 과정(reasoning)**이 표시되어서 왜 그런 결정을 내렸는지 투명하게 확인할 수 있습니다.
>
> (ActivityLog) 스케줄러에 의한 자동 실행 기록과 온체인 이벤트 로그도 확인 가능합니다.

### 6-4. 위임 설정 (`/agent/delegate/[id]`) — 핵심 시연

#### 화면에 보여줄 것
- **시나리오 선택** (Morpho / Liquity 토글)
- **위임 상태 카드** — 현재 권한 부여 상태
- **3-Step 위저드**:

  **Step 1: Vault 입금**
  - 토큰 선택 (sbUSD, wCTC, lstCTC, USDC)
  - 현재 Vault 잔액 표시
  - 입금/출금

  **Step 2: 권한 부여**
  - 프리셋 스코프 (허용할 함수 목록)
  - 만료일: 30일
  - 토큰 한도 표시 (예: sbUSD 1000개)
  - Grant Permission / Revoke

  **Step 3: 프로토콜 위임**
  - Liquity: Trove ID 입력 → setAddManager + setInterestIndividualDelegate
  - Morpho: setAuthorization
  - Revoke 버튼

#### 대본

> 이것이 Agent 시스템의 핵심 — **위임 설정 위저드**입니다. 3단계로 진행됩니다.
>
> (시나리오 선택) 먼저 어떤 프로토콜에 대해 위임할지 선택합니다. Liquity CDP Agent를 설정해보겠습니다.
>
> **Step 1 — Vault 입금**: Agent가 사용할 자금을 전용 Vault에 입금합니다. 이 Vault는 AgentVault 컨트랙트로, 사용자 자금을 안전하게 격리합니다. Agent는 이 Vault 안의 자금만 사용할 수 있습니다.
>
> **Step 2 — 권한 부여**: Agent에게 어떤 행동을 허용할지 정의합니다. 예를 들어 Liquity Agent는 `adjustTroveInterestRate`(이자율 조정)과 `addColl`(담보 추가)만 허용됩니다. 토큰 한도도 설정되어서 Agent가 무제한으로 자금을 쓸 수 없습니다. 만료일은 30일이고, 언제든 Revoke로 즉시 권한을 회수할 수 있습니다.
>
> **Step 3 — 프로토콜 위임**: 마지막으로 실제 프로토콜에 Agent를 등록합니다. Liquity에서는 두 가지 권한이 필요합니다 — AddManager(담보/부채 관리 권한)와 InterestDelegate(이자율 조정 권한, 최소/최대 범위 제한 포함). 이 두 권한을 모두 부여해야 완전한 위임이 완료됩니다.
>
> (상태 카드) 위임 상태가 실시간으로 업데이트되어서 현재 어떤 권한이 활성화되어 있는지 항상 확인할 수 있습니다.

### 6-5. Agent Vault (`/agent/vault`)

#### 화면에 보여줄 것
- 지원 토큰별 잔액 목록
- Deposit/Withdraw 버튼
- PermissionList — 활성 권한 목록 (만료일, 토큰 한도)

#### 대본

> Agent Vault 페이지에서는 Vault에 예치된 자금과 활성 권한을 종합적으로 관리합니다.
>
> 토큰별 잔액을 확인하고 추가 입금이나 출금이 가능합니다. 하단의 Permission List에서는 현재 부여된 모든 Agent 권한, 만료일, 토큰 한도를 한눈에 볼 수 있습니다.

## 아웃트로 (1분)

### 대본

> 지금까지 Snowball Protocol의 전체 모듈을 시연했습니다.
>
> 정리하면:
> - **DEX** (Uniswap V3) — 집중화 유동성 기반 토큰 교환 + LP 관리
> - **CDP** (Liquity V2) — 거버넌스 없는 담보 대출, 유저 설정 이자율
> - **Lending** (Morpho Blue) — 모듈러 독립 마켓 대출
> - **Yield** (Beefy) — 자동 복리 수익 볼트
> - **Bridge** (Wormhole + Worker) — 크로스체인 자산 이동
> - **Agent** (ERC-8004) — 온체인 AI Agent로 자동화된 포지션 관리
>
> 이 모든 것이 하나의 체인, 하나의 프로토콜에서 통합 운영됩니다. 각 모듈이 독립적이면서도 서로 연결되어 — DEX의 유동성 위에 CDP가 스테이블코인을 발행하고, 그 스테이블코인이 Lending과 Yield에서 활용되며, Agent가 이 모든 포지션을 자동으로 관리합니다.
>
> 감사합니다.
